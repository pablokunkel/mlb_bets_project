## What's actually broken

### 1. Park-archetype Phase 2 backfill produces garbage

`features_v2.compute_batter_park_archetype` JOINs `batter_hr_events`
against `daily_slate` for venue lookup. `daily_slate` is live-only —
only ~371 rows covering 2026-04-29 → 2026-05-25. The historical 2024-25
HRs don't have venue data. **Yesterday's run wrote 114,495 rows but
ALL with NULL `feature_centroid_json`.** Phase 2 looks done in `tasks`
but actually wrote no usable data.

**Fix path:** build a `game_pk → venue` lookup either by:
- one-off ETL hitting MLB Stats API for the ~4,761 historical game_pks
  that don't have `daily_slate` rows, OR
- enriching `batter_hr_events` with `home_team` at Statcast ETL time
  (Statcast already returns this field), then mapping `home_team →
  venue` via a small static table.

Second option is cheaper. ~4,761 API calls × ~200ms = ~16 min for
option 1; static map = seconds for option 2.

### 2. Form-archetype Phase 2 builder still has NA bug

The cached parquet bulk pull is still on disk → next run skips the
25-min download.

## Hard-won lessons from the previous session

- **Don't trust agent prose summaries over tool-log data.** Agents
  inflated their runtime estimates and overstated finding severity.
- **Verify against the user's actual environment.** Python 3.14 +
  Windows + PowerShell. Two bugs (argparse %, NA on parquet round-trip)
  slipped through agents testing on different Python versions.
- **Backfill scripts that query live-only tables for historical dates
  will silently produce empty results.** Verify every `FROM <table>`
  in a backfill script against actual DB date coverage before running.
- **Carrying agent worktrees on Windows leaves orphan locks.** Clean
  up with `Get-ChildItem .git\worktrees | Where-Object { $_.Name -notin
  @(keep,list) } | Remove-Item -Recurse -Force -ErrorAction
  SilentlyContinue` periodically.
- **`merge` ≠ `lands on main`.** If a PR's base is `claude/<something>`
  instead of `main`, the merge button silently merges into that branch.
  Two PRs hit this last session (#84 form, #87 park) and required
  carry-forward PRs.
